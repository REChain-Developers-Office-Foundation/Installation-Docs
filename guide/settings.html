# Cronjobs

To add applications from the Google Play Store and Apple App Store, you will need to run cronjob links at specific intervals. It is recommended to run these links at least once every 15 minutes. Please be aware that running them more frequently could result in sending an excessive number of requests to the respective app markets and may lead to your server's IP address being blocked. In such a situation, you might need to consider changing your server.

To run cronjob links, you can use your hosting panel's cron jobs feature, which is available in most modern hosting panels. You can find an example cURL command right below the cronjob link.

  • Google Play Store Cronjob Link: The URL you need to execute to download apps from the Google Play Store.
  • Apple App Store Cronjob Link: The URL you need to execute to download apps from the Apple App Store.
  • Cronjob Link (System): To clear the system query cache (enhancing site speed) and reset vote records (preventing multiple votes), run the Crontab Link periodically (1-2 times per day recommended).
  • Cronjob Link (Hourly): To keep the 'Popular Apps In Last 24 Hours' section up to date, run this Crontab Link every hour.

TIP

You can click 'Generate New Link' to generate a new link. Please be aware that creating a new link will disable the old one.

# Content Scraper Settings

  • Submission Platform (Google): Please select the platform where apps will be added after being retrieved from the Google Play Store.
  • Submission Platform (Apple): Please select the platform where apps will be added after being retrieved from the Apple App Store.
  • Google Play Country: Please specify the country code for the Google Play Store from which apps will be added. The code should be 2 characters. Example: DE for Germany, US for the United States. You can find a list of "ISO 3166-1 alpha-2 Country Codes" here: https://en.wikipedia.org/wiki/ISO_3166-1_alpha-2
  • Google Play Language: Please specify the code for your preferred language in the Google Play Store. For example, if you choose 'de' (German), the content manager will search for applications in the German language. Please be aware that not all applications have translations available in local languages as it depends on the application publishers. You can find a list of supported languages here: https://support.google.com/googleplay/android-developer/table/4419860
  • Apple App Store Country: Please specify the country code (ISO 3166-1 alpha-2) for the Apple App Store from which the applications will be retrieved. The code should be 2 characters. You can find a list of supported countries here: https://support.apple.com/en-us/HT204411 To check the 2-character country code, please click here: https://en.wikipedia.org/wiki/ISO_3166-1_alpha-2

WARNING

Please note that the country code should be in lowercase, for example, 'us' for the USA, and 'de' for Germany, when retrieving apps from the Apple App Store.

  • Screenshots to be Downloaded: Please specify the number of screenshots you want to download when retrieving and adding apps from the respective app markets.
  • Google Play Store Auto Submission: You can change whether apps from the Google Play Store will be added automatically. Please ensure that you have configured the cron job.
  • Apple App Store Auto Submission: You can change whether apps from the Apple App Store will be added automatically. Please ensure that you have configured the cron job.

# SEO Settings

On the SEO Settings page, you can configure specific settings for each chosen country. This allows you to tailor your website's SEO settings for different regions or locales.

On this page, you can customize the following elements:

  • Site Title: Modify the title of your website in the chosen language
  • Site Description: Modify the description of your website in the chosen language
  • H1 Title: If this field is filled, a text using the H1 tag in the chosen language will be added to the homepage.

Additionally, you can adjust permalinks. Permalinks are the permanent URLs that point to your website's content. They can be customized to improve SEO and user experience.

# OpenAI Settings

The OpenAI Settings page allows you to configure various parameters related to content generation and regeneration. These settings include:

  • OpenAI Max Tokens: This parameter allows you to control the maximum number of tokens used in the generated content, affecting its length and complexity.
  • OpenAI Temperature: The temperature setting influences the randomness of content generation. Higher values increase randomness, while lower values make the content more deterministic.
  • OpenAI Content Regeneration Command: Here, you can write the command that will be sent to OpenAI when you click the "Regenerate via OpenAI" button. You can specify the App Title using the %content% short tag.

Example App: WhatsApp

Example Command: Write an article related to %content%.

The output of this command: Write an article related to WhatsApp.

  • OpenAI Content Regeneration Command (Auto Submission): It's the same as the previous one, but the command is sent to OpenAI in auto-submission mode
  • Content Regeneration in Auto Submission Mode: In this section, you can determine whether the content within the app will be regerated in auto submission mode.

# CDN Settings

On this page, you can upload basic asset files to CDN by clicking the "Push assets to the CDN" button.

The following files will be pushed to the CDN:

  • /images/logo.png
  • /images/pixel.png
  • /images/favicon.png
  • /images/no_image.png
  • /images/default_share_image.png

If you have previously hosted assets such as app, app screenshots, news, topics images etc. on your server, you need to manually upload them to the CDN. In the next version, these will be done automatically and detailed CDN settings will be added to the panel. (Settings like which files to run from the CDN.)

After configuring the CDN settings in the .env file, the system will automatically upload new assets such as app, app screenshots, news, topic images, and more to the CDN.

WARNING

Please note that the CDN feature is a new feature, if you experience any problems, please inform us.

For CDN configuration, please edit the .env file and change the lines below according to the CDN service you want to use.

Step 1 - Change the File System

FILESYSTEM_DRIVER=file

to

FILESYSTEM_DRIVER=s3

or

FILESYSTEM_DRIVER=wasabi

Step 2 - Add Security Credentials

  • Amazon S3
AWS_ACCESS_KEY_ID=
AWS_SECRET_ACCESS_KEY=
AWS_DEFAULT_REGION=
AWS_BUCKET=
AWS_USE_PATH_STYLE_ENDPOINT=false
  • Wasabi S3
WAS_ACCESS_KEY_ID=
WAS_SECRET_ACCESS_KEY=
WAS_DEFAULT_REGION=
WAS_BUCKET=
WAS_URL=
  • BunnyCDN
WAS_BUNNY=0
WAS_BUNNY_ZONE=

If you want to use Wasabi with BunnyCDN, please change WAS_BUNNY to 1.

WAS_BUNNY=1

# Sitemap Settings

A sitemap is essentially a structured list of all the pages and resources on your website, making it easier for search engines to crawl and index your content.

Here, you can access the sitemap index URL, which serves as a centralized gateway for search engines to discover and index your website's content efficiently.

Additionally, on this page, you have the ability to customize essential sitemap settings, including:

  • Change Frequency: Specify how often different pages on your website are updated. This information helps search engines understand how frequently they should revisit and refresh their indexed content.
  • Priority Levels: Assign priority levels to different pages to signal their importance. This guides search engines in prioritizing which content to crawl and index first.

Furthermore, you have the flexibility to add additional links to the sitemap, ensuring that even the most specific content is included in the sitemap's structure.

# PWA Settings

Progressive Web App (PWA) is a modern web technology that combines the best features of traditional websites and mobile applications. PWAs offer a fast, responsive, and user-friendly experience similar to native mobile apps.

  • Enable PWA: By enabling this option, you can activate Progressive Web App functionality on your website.
  • PWA Name: Define a name for your Progressive Web App.
  • PWA Short Name: Choose a shorter, concise name for your PWA.
  • PWA Description: Provide a brief description of your Progressive Web App.
  • PWA Theme Color: Fill the theme color that represents your PWA. This color will be displayed in the address bar and task switcher.
  • PWA Background Color: Fill the background color that complements your PWA's theme.
  • PWA Screenshot: Upload a screenshot with dimensions 1080x1920.
  • PWA Icons: You need to upload four PWA icons with the following dimensions: 512x512, 192x192, 48x48, 24x24

TIP

Working on many different browsers is a significant advantage of PWA over web apps. This strength works perfectly fine on Android, whether Chrome, Firefox, or Microsoft Edge. However, with iOS, only Safari supports PWA.

Unlike Windows and Android users, iOS users often prefer to use Safari to access the Internet. PWA still works on Safari, yet not all features are supported.

# Translations

Here, you can add new languages to your website or edit existing ones, and rearrange their order. To change the order of a language, simply drag it up or down.

  • Language: Name of the language.
  • Language Code: 2-letter code of the language, for example, for the German language: de. You can find a list of "ISO 3166-1 alpha-2 Country Codes" here: https://en.wikipedia.org/wiki/ISO_3166-1_alpha-2
  • OG:locale Tag: OG:locale code of the language. You can find a list of "List of Country Locale Codes" here: https://saimana.com/list-of-country-locale-code/
  • Country Flag: Choose a flag from the list.
  • Text Direction: Specify whether the text direction is Left-to-Right (LTR) or Right-to-Left (RTL).
