# Apps

# Create App

  • Title: The name of the application.
  • Slug: A user-friendly URL identifier for the application.
  • Image: The visual representation or logo of the application.
  • Description: A brief summary or explanation of the application's purpose and features.
  • Tags: Keywords associated with the application (use commas to separate).
  • Package Name: The unique identifier for the application package.
  • Categories: The specific categories the application belongs to.
  • Platforms: The platforms supported by the application.
  • Button Type: The label or type of action button.
  • Developer: The individual or company responsible for developing the application.
  • License: Information about the application's licensing (e.g., Free or $0.99).
  • Buy URL: A link to purchase or download the application.
  • Page Views: The total number of page views for the application.
  • Up Votes: The number of positive votes or ratings received.
  • Down Votes: The number of negative votes or ratings received.
  • Featured: This is intended for apps that are meant to be featured in the Featured Apps section on the homepage.
  • Pinned: This is used to place the app at the top of the category page and within the category section on the homepage.
  • Editor's Choice: This is meant for apps that are intended to be showcased in the Editor's Choice section.
  • Must-Have: This is for apps that are intended to be highlighted in the Must-Have Apps section on the homepage.
  • Content: Content or information about the application.

# AI Features

  • Generate New Content via OpenAI: You can instruct OpenAI to write content on a specific topic. For example: "Write an article about Mobile Application security tips"
  • Regenerate via OpenAI: This section becomes active after adding the app, allowing you to recreate the content using the OpenAI API.

# Version Details

  • Version: The current version of the application.
  • File Size: The size of the application file.
  • Downloads: The total number of downloads for the application.
  • File: The application file itself.
  • Download/Page URL: The URL for downloading the application or visiting the page.

If you wish to upload a file, click on "Browse" in the "File" section to select the file. In this case, there is no need to enter any link in the "Download/Page URL" field; the download link will be displayed in this area after the file is uploaded. The "Download/Page URL" section is intended for adding a download link when you do not want to upload a file.

# Translations

This section becomes active after adding the app, allowing you to translate any application you desire or delete the translations you've added. This section utilizes the Google Translate infrastructure. Excessive requests may temporarily disable this feature. If you encounter such an issue, please retry after some time.

# SEO Settings

  • Custom Meta Title: A custom title for SEO metadata.
  • Custom Meta Description: A custom description for SEO metadata.
  • Custom H1 Title: A custom H1 title for SEO optimization.

This section is designed for customizing specific SEO elements, including meta title, meta description, and H1 tags. You can use the following supported shortcodes to dynamically generate these elements:

  • %app_title% Inserts the title of the application.
  • %categories% Displays the categories associated with the application.
  • %platforms% Shows the supported platforms for the application.
  • %site_title% Represents the title of the website.
  • %sep% Adds a separator between elements.
  • %year% Displays the current year.
  • %month% Displays the current month in numeric format.
  • %day% Displays the current day of the month.
  • %month_text% Shows the name of the current month.
  • %day_text% Displays the name of the current day of the week.

# Screenshots

This section becomes active after adding the app, allowing you to upload screenshots and other visuals related to the app.

# RSS Feeds

The system comes with RSS feed support. These are some example RSS feed addresses that the system supports:

https://www.yoursite.com/rss

This URL represents the main RSS feed.

https://www.yoursite.com/de/rss

This URL represents a customized RSS feed for the German language.

https://www.yoursite.com/rss/category/games

This URL represents an RSS feed that specifically includes content from the games category.

https://www.yoursite.com/de/rss/category/games

This URL represents a customized RSS feed in the German language, providing content from the games category.

# OpenAI API

To harness the power of AI features, you'll need an OpenAI API. You can create your OpenAI API by clicking here. Once you've generated your API, you'll need to add it to the OPENAI_KEY field at the bottom of the .env file located in the lapp directory.

When encountering error messages related to the use of AI features, please reach out to OpenAI support directly. We do not have the capability to provide solutions for such issues.
